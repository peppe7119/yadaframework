YadaJob¶

public abstract class YadaJob implements Callable<Void>¶

The base class for jobs handled by the YadaScheduler. Subclasses must implement the run() method. Uses joined inheritance so that subclasses have their own table; the subclass id, which must not be declared in java but exists in the table, gets the same value as the YadaJob id.

jobRecoverable¶

protected boolean jobRecoverable¶

Tell if a job should be started immediately again from the beginning after a system crash/shutdown while running. When true, the job should take care of deleting or skipping any partial results, if needed. When false, the job will become DISABLED after a system crash (true by default)

jobsMustBeActive¶

protected List<YadaJob> jobsMustBeActive¶

Run the current job only while the jobMustBeActive instance is active. If that instance is deleted, the current job is deactivated

jobsMustBeInactive¶

protected List<YadaJob> jobsMustBeInactive¶

Run the current job only if the jobsMustBeInactive instance is not active/paused nor running. If that instance is deleted or deactivated, the current job can be activated

jobsMustComplete¶

protected List<YadaJob> jobsMustComplete¶

Run the current job after the jobMustComplete completes successfully.

recovered¶

protected boolean recovered¶

True when the current invocation is on a job that was running when the server crashed and has the jobRecoverable flag true. Implementations can choose what to do when recovering from a crash.
